# 安全编码规范

本文件只维护RAG 知识检索服务安全规则。AI 新增或修改接口、报告内容、生成任务、导出、分享、文件、日志、脱敏和测试时，必须先阅读本文件。

## 接口鉴权

- 新增或修改 Controller 时，必须明确接口是否需要登录和授权；除健康检查等明确公开接口外，默认必须加 `@PreAuthorize`。
- 管理端接口权限码必须与 `user` 项目 `auth_resource` 中的后端资源一致，不能只依赖前端隐藏菜单或按钮来保护接口。
- 报告生成、批量导出、审核发布、分享访问、导入导出等高风险操作必须有明确权限码、租户校验和测试覆盖。
- 登录态相关接口如果出现在RAG 知识检索服务中，必须沿用统一认证上下文；退出登录、用户禁用或密码变更后的 token 清理和失效规则必须与 `user` 服务保持一致。
- 退出清理 token 是统一认证体系要求，RAG 知识检索服务不得自行绕过或保留可继续访问报告接口的旧 token。
- 公开回调、健康检查或内部调用接口必须说明公开原因、来源校验方式、签名校验方式和限流策略。
- 报告生成、批量导出、文件预览、分享访问、导出记录等接口必须增加频率限制、数量限制和审计日志，防止资源滥用和业务流程滥用。
- 如果接口使用 Cookie、浏览器自动携带凭证或管理后台表单提交，必须评估 CSRF 风险；高风险写操作必须使用 SameSite、CSRF Token、Origin/Referer 校验或改为 Bearer Token 模式。

## 水平越权和数据隔离

- 查询、详情、修改、删除接口必须校验租户上下文和数据归属，防止用户通过替换 `id`、`tenantId`、`ownerUserId`、`deptId` 访问或修改他人数据。
- 不允许把前端传入的角色、权限、租户、用户归属直接作为可信事实；授权关系必须从数据库和 `UserContextHolder` 当前认证用户推导或校验。
- 用户报告查询必须按当前认证用户、报告归属和可见范围过滤；管理端查询用户报告时必须走管理权限和租户隔离。
- 报告、报告文件、生成任务、审核记录、分享链接和附件必须防止水平越权；只校验用户已登录不等于有权读取目标报告或目标附件。
- 批量生成、批量导出、批量删除、批量审核等批量操作必须逐条校验租户、用户归属和目标范围。
- BO、DTO、Query 入参必须使用允许写入字段的白名单对象，禁止把 Entity 直接作为请求体，防止批量赋值修改 `tenantId`、`state`、`isDelete`、`ownerUserId`、`sendStatus` 等敏感字段。
- 返回 VO、导出记录和通知回执必须执行字段级授权，普通用户不得读取其他用户报告、手机号、邮箱、外部通知目标、失败原因详情等敏感属性。

## 报告内容安全

- 报告标题、摘要、正文、备注和文件预览内容默认按纯文本处理；如果允许 HTML，必须做白名单清洗，防止 XSS 跨站脚本。
- 报告生成变量只能替换经过校验的数据，不允许把用户输入当成表达式、脚本或 SQL 片段执行。
- 批量生成、导出或分享必须限制目标范围和数量，避免越权覆盖其他租户、其他业务线或全量用户。
- 用户昵称、标题、内容、生成变量、跳转链接、文件名等可回显字段必须防止 XSS，不得把未清洗输入拼接进 HTML、Markdown、JavaScript、URL 或响应头。
- 外部跳转链接必须限制协议和域名白名单，禁止 `javascript:`、`data:` 等可执行协议。
- 报告渲染不得执行不可信表达式、脚本、反射调用或任意方法；确需表达式时必须限制变量、函数、执行时间和可访问类型。
- 报告链接、图片链接、Webhook、回调地址必须防止 SSRF，不能让用户输入触发服务端访问内网、本机、云元数据或基础设施地址。

## SQL 注入和查询安全

- Service 查询必须优先使用 MyBatis-Plus Wrapper、LambdaWrapper 或参数绑定，不允许拼接用户输入形成 SQL。
- 确需动态字段排序或字段选择时，必须使用白名单校验字段名。
- 查询对象中的 `fields`、`collationFields`、`query` 等可控参数必须限制用途，不能直接拼接成任意 SQL 片段。
- XML Mapper 中使用 `${}` 时必须先证明参数来自后端白名单；用户输入只能走 `#{}` 参数绑定。
- 分页、排序、关键字、报告类型、生成状态、审核状态、时间范围等查询参数必须限制字段、方向、长度和范围。
- 查询接口必须限制 `pageSize`、导出条数、时间跨度、模糊查询长度和批量 ID 数量，防止无限制资源消耗。

## SSRF 和外部请求

- 后端如果根据用户输入访问 URL、Webhook、回调地址、图片地址、文件地址或第三方接口，必须做协议、域名、端口、IP 段和重定向后的最终地址白名单校验。
- 禁止访问内网地址、本机地址、云元数据地址、Nacos、Redis、MySQL、Actuator、Seata、网关管理端等内部资源。
- 外部请求必须设置连接超时、读取超时、最大响应体、重定向限制和失败降级，不允许把用户输入直接作为完整目标 URL。

## 加密、密钥和敏感配置

- 短信、邮件、推送、Webhook、对象存储、MQ、Redis 等密钥必须来自 Nacos、环境变量或密钥管理系统，不得写入代码、README 示例真实值或测试数据。
- AI 发现已有配置、脚本、SQL、测试或文档中存在疑似密钥、RabbitMQ 地址、Nacos 地址、数据库连接或生产配置时，只能报告文件和行号并说明风险，不得直接替换为占位符、删除或迁移。
- 新增密钥、通知通道凭证、内部地址或生产连接信息必须先征得项目负责人确认。
- 签名、验证码、临时链接、退订链接、阅读确认链接必须具备过期时间、一次性或可撤销能力，不能出现在日志、URL 查询串或异常信息中。
- 加密、签名、随机数和验证码必须使用 JDK 或成熟库的安全实现，不得手写加密算法或使用可预测随机数。

## 敏感字段加密解密与脱敏流程

- 报告归属人手机号、邮箱、外部账号、身份证号、详细地址、通知目标和报告内容中的个人信息默认按“前端提交明文、后端校验、入库前加密、数据库保存密文、查询后仅在后端内存解密、普通接口返回 VO 脱敏值”的链路处理。
- 加密解决数据库泄露风险，脱敏解决接口、页面、通知回执和日志泄露风险；两者不能互相替代。
- 普通列表、详情、生成记录、通知回执、导出和日志接口只能返回脱敏值，不得返回完整手机号、完整邮箱、完整外部账号或完整个人身份信息。
- 查看完整明文必须设计独立接口或独立动作，不能复用普通详情接口，也不能通过 query 参数控制普通接口返回明文。
- 明文查看接口必须具备专门权限码、登录校验、租户校验、数据权限校验、字段级授权、操作理由、审计日志和必要的二次确认或二次认证。
- 明文查看接口返回结果不得缓存，不得写入浏览器地址、日志、异常、审计详情、报告正文、通知内容或第三方通知。
- 编辑报告归属人、分享对象或生成变量时默认不回显明文；前端显示脱敏值，未修改时不提交该字段，修改时要求重新输入完整值。
- 如业务确实需要按手机号、邮箱、外部账号查询报告归属人或分享对象，必须设计标准化 hash、盲索引或专用检索字段；禁止直接对密文字段做普通 `like` 查询。

## 脱敏和日志

- 返回 VO 必须做数据脱敏，不向前端返回密码、token、密钥、验证码、手机号完整值、邮箱完整值、内部地址、异常堆栈等敏感信息。
- 日志、异常、审计记录不得输出密码、token、密钥、验证码、完整手机号、完整邮箱、数据库连接串和内部服务地址。
- 报告正文可能包含个人信息，日志只记录报告 ID、任务编码、处理结果和必要追踪字段。
- 数据脱敏必须在 VO 组装、序列化或统一出参层完成，不能只依赖前端隐藏字段。
- 导出、下载、推送回执、审计详情等非页面接口同样必须执行脱敏规则。

## 文件遍历和上传校验

- 文件读取、导入、导出和下载功能必须防止路径遍历；用户输入的文件名只能作为文件名使用，不能直接参与目录拼接，必须校验规范化路径仍在允许目录内。
- 文件上传必须校验大小、扩展名、MIME 类型和实际文件头；禁止上传脚本、可执行文件、HTML、SVG 等可能导致执行或 XSS 的内容，文件名必须重新生成。
- 文件保存目录必须由服务端配置决定，不能由请求参数决定；下载报告附件必须校验当前用户或当前租户是否有权访问附件记录。
- 解压、批量导入、报告模板导入必须防止 Zip Slip、超大文件、空文件、重复文件名覆盖和伪造后缀。
- 上传文件不得直接放在可执行目录或静态公开目录；需要公开访问时必须通过受控下载接口、签名 URL 或对象存储权限策略。
- 解析 Excel、PDF、图片、压缩包等复杂文件时必须限制解析深度、页数、尺寸、行数和内存占用。

## 安全配置和供应链

- Spring Security、CORS、Actuator、OpenAPI、错误页、静态资源、上传大小、请求体大小等配置必须按最小暴露原则开启，生产环境不得默认开放调试入口。
- 新增依赖、插件、镜像、脚本和 Gradle/Maven 仓库时必须说明用途，优先使用可信仓库和稳定版本，避免引入未维护、漏洞严重或来源不明的组件。
- 依赖升级必须关注安全公告和破坏性变更，不得为了追最新版本跨大版本引入不兼容风险。
- CI/CD、SQL 脚本、初始化数据和配置变更必须保持可审计、可回滚，不能在构建脚本中明文写入密钥。

## 异常处理、完整性和反序列化

- 异常处理必须失败关闭，权限、租户、签名、验证码、token 校验异常不得降级为允许生成、允许读取或允许导出。
- 不得反序列化不可信 Java 对象、脚本、表达式、模板或规则；报告生成变量只能作为数据参与渲染。
- 回调、Webhook、通知回执、第三方事件、导入数据必须校验签名、时间戳、幂等键和数据完整性，不能只按请求来源 IP 或参数存在判断可信。
- 关键写操作必须具备幂等、防重复提交和审计记录，避免重放请求造成重复生成、重复审核或重复回调处理。

## 安全日志、告警和审计

- 登录失败、权限拒绝、跨租户访问、越权访问、token 失效、敏感数据导出、批量生成、审核发布、异常上传、SQL 注入和 XSS payload 命中必须记录安全审计日志。
- 安全日志必须包含时间、租户、用户、接口、目标资源、结果、原因和 traceId，但不得包含敏感明文、完整报告正文或完整归属人信息。
- 高风险安全事件必须具备告警或可检索标识，不能只写普通业务日志后无人关注。

## 测试要求

- 新增接口测试时，必须覆盖未登录、无权限、跨租户或替换他人 `id` 的失败场景；只测成功路径不算完成安全验证。
- 报告生成测试必须覆盖无权限生成、跨租户目标、非法生成变量和批量数量限制。
- 用户报告测试必须覆盖当前用户只能读取自己有权访问的报告，管理端必须具备管理权限才能查询他人报告。
- 文件上传下载测试必须覆盖非法扩展名、超限大小、伪造 MIME、路径穿越文件名和空文件等场景。
- 查询测试必须覆盖排序字段白名单、SQL 注入 payload、超长关键字和非法时间范围。
- 认证测试必须覆盖退出登录后旧 token 不可继续访问，密码变更或用户禁用后历史 token 按当前设计失效。
- 安全测试还必须覆盖 CSRF、SSRF、限流、批量赋值、字段级越权、异常失败关闭、敏感配置泄露、依赖风险、安全日志审计和报告业务流程滥用。
