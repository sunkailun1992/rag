# 错误码规范

本文件整理 `utils` 中 `com.kellen.utils.enumeration.ReturnCode` 的错误码，供 AI 编码时选择统一返回码。业务代码优先复用已有 `ReturnCode`，确实缺少语义时再扩展公共枚举和本文件。

## 使用原则

- 成功统一使用 `ReturnCode.成功`，返回码为 `0`。
- 用户输入、登录、权限、重复提交、客户端版本、隐私授权等问题归入 `A` 类。
- 当前系统业务逻辑、程序执行、系统超时、限流、资源耗尽等问题归入 `B` 类。
- RPC、消息、缓存、配置、网关、数据库、通知等外部或基础设施依赖问题归入 `C` 类。
- Controller 最外层返回必须使用 `ApiResponse`，成功返回推荐 `ApiResponse.success(data)`。
- 业务失败优先抛出带 `ReturnCode` 的业务异常，或返回 `ApiResponse.fail(returnCode, "稳定错误提示")`。
- 对前端展示的错误信息必须稳定、清晰，不直接暴露 SQL、堆栈、密钥、token、内部服务地址等敏感信息。
- 参数校验失败优先使用 `A0410`、`A0421`、`A0427` 等用户请求参数错误码，不使用 `B0001`。
- 无登录态、登录过期、权限不足分别优先使用 `A0220`、`A0230`、`A0301` 或 `A0312`。
- 幂等或重复提交使用 `A0506`。
- MyBatis-Plus 乐观锁更新失败属于并发写入冲突，不再按 HTTP 请求头 API 版本号处理；项目有专用并发错误码时优先专用码，没有时使用稳定业务失败提示。
- 数据库表、列、死锁、主键冲突等基础设施问题归入 `C03xx`，不要包装成普通用户参数错误。

## 选码优先级

| 场景 | 推荐错误码 | 说明 |
| --- | --- | --- |
| 成功 | `0 成功` | Controller 成功统一使用 |
| 用户名或密码错误 | `A0210 用户密码错误` | 不暴露账号是否存在时也可使用该码 |
| 用户不存在 | `A0201 用户账户不存在` | 只有业务允许暴露时使用 |
| 用户禁用或冻结 | `A0202 用户账户被冻结` | 登录和敏感操作均适用 |
| 未登录或身份缺失 | `A0220 用户身份校验失败` | 当前用户上下文为空 |
| 登录过期 | `A0230 用户登录已过期` | token 或会话过期 |
| 未授权 | `A0301 访问未授权` | 未取得访问授权 |
| 权限不足 | `A0312 无权限使用` | 已登录但缺少权限 |
| 必填参数缺失 | `A0410 请求必填参数为空` | `null`、空字符串、缺少字段 |
| 参数格式错误 | `A0421 参数格式不匹配` | 类型、格式、枚举值不匹配 |
| JSON 解析失败 | `A0427 请求JSON解析失败` | 请求体不是合法 JSON |
| 内容非法或敏感词 | `A0430 用户输入内容非法` 或 `A0431 包含违禁敏感词` | 文本、图片、文件内容校验 |
| 重复提交 | `A0506 用户重复请求` | 幂等锁命中 |
| 上传文件类型错误 | `A0701 用户上传文件类型不匹配` | 文件后缀、MIME、业务类型不符 |

| 系统超时 | `B0100 系统执行超时` | 当前系统处理超时 |
| 系统限流 | `B0210 系统限流` | 服务自身限流 |
| RPC 失败 | `C0110 RPC服务出错` | 调用远程服务失败 |
| 缓存失败 | `C0130 缓存服务出错` | Redis 等缓存异常 |
| 配置失败 | `C0140 配置服务出错` | Nacos 等配置服务异常 |
| 数据库失败 | `C0300 数据库服务出错` | SQL 执行、连接、驱动等异常 |
| 主键冲突 | `C0341 主键冲突` | 数据库唯一键或主键冲突 |
| 通知失败 | `C0500 通知服务出错` | 短信、语音、邮件等通知失败 |

## 扩展规则

- 新错误码必须先确认现有 `ReturnCode` 是否已覆盖语义。
- 扩展时同时修改 `utils` 的 `ReturnCode` 枚举和本文件，保持文档与代码一致。
- 错误码命名使用稳定业务含义，不使用临时需求名、接口名或页面文案。
- 同一类错误优先复用大类码，不为每个接口新增碎片化错误码。
- 用户侧可修复的问题用 `A`，服务端需要排查的问题用 `B`，外部依赖或基础设施问题用 `C`。
- 不要复用已经存在的编码，不要改变既有编码含义。

## 异常处理建议

| 异常或失败来源 | 推荐处理 |
| --- | --- |
| `UserException` 且携带 `ReturnCode` | 直接返回异常携带的 `ReturnCode` |
| 参数绑定或校验异常 | 返回 `A0410` 或更精确的 `A0421` |
| 幂等异常 | 返回 `A0506` |
| 业务规则不满足 | 优先选择 `A04xx`、`A03xx` 或已有业务语义码 |
| 当前系统执行异常 | 返回 `B0001`，日志保留上下文 |
| RPC 调用异常 | 返回 `C0110` 或 `C0210` |
| 短信、邮件、语音通知异常 | 返回 `C0500`、`C0501`、`C0502` 或 `C0503` |
| 数据库异常 | 返回 `C0300`，死锁和主键冲突使用更精确码 |

| `0` | 成功 |
