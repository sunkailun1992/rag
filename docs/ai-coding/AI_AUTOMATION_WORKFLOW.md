# AI 自动化开发流程

本文用于约束 AI 在任意业务需求中按统一流程完成需求理解、方案拆解、工程实现、验收验证和交付说明。本文只保留通用流程，不写某个微服务的私有接口、默认账号、私有 IP、真实 token 或本机绝对路径。

## 一、标准流程

1. AI 需求说明：先明确业务背景、调用方、模块边界、接口入口、权限、数据库、租户、数据权限、敏感字段和不做范围。
2. AI 验收标准：在开发前列出接口、数据库、权限、安全、异常、测试、回归和发布验收点。
3. AI 开发手册：写清 Controller、BO、Query、VO、Service、ServiceQuery、ServiceResults、Mapper、Entity、SQL、权限码、配置和测试入口。
4. AI 工程实现：先数据模型，再查询对象，再服务层，再接口层，再 SQL、权限和测试；每一步都遵守当前项目分层和注释规范。
5. AI 测试说明：说明执行过的编译、单元测试、请求层测试、SQL 验证、权限验证和联调验证。
6. AI 提交说明：说明改动范围、影响接口、影响表、权限码、配置项、兼容影响、回滚方式和未覆盖风险。
7. 回归发布说明：说明 SQL 执行顺序、配置刷新、服务重启、菜单权限刷新、数据修复和回滚策略。

## 三、验收标准模板

```markdown
# AI 验收标准 - <需求名称>

## 接口
- [ ] 新增接口参数校验正确。
- [ ] 修改接口只更新允许字段。
- [ ] 删除接口符合软删或硬删要求。
- [ ] 查询接口支持分页、排序、字段白名单和查询条件。
- [ ] 接口路径、HTTP 方法和请求体符合 RESTful 规范。

## 数据库
- [ ] 表字段、索引、默认值、唯一约束正确。
- [ ] 租户字段处理正确。
- [ ] 逻辑删除字段处理正确。
- [ ] 乐观锁 version 处理正确。
- [ ] DDL 脚本按 ddl_history 规则新增或修改。
- [ ] 全新或空业务库首次启动前，已在目标业务库手动执行 `../utils/src/main/resources/db/common-infra-schema.sql`。

## 权限
- [ ] Controller 权限码正确。
- [ ] 菜单、按钮或后端资源权限存在。
- [ ] 数据权限规则覆盖本人、部门、部门树或自定义范围。
- [ ] 明文查看敏感字段有独立权限、数据权限和审计日志。

## 安全
- [ ] 未登录访问被拒绝。
- [ ] 无权限访问被拒绝。
- [ ] 跨租户访问被拒绝。
- [ ] 替换他人 id 或 tenantId 无法越权。
- [ ] SQL 注入、XSS、文件遍历、批量赋值和资源消耗风险已检查。

## 敏感信息
- [ ] 手机号、邮箱、身份证、地址、联系人、token、apiKey、openId、unionId、签名 URL 等敏感字段已识别。
- [ ] 入库前完成加密、hash 或盲索引辅助字段处理，数据库不新增明文敏感字段。
- [ ] 普通列表、详情、导出、日志、异常、MQ 消息只返回或记录脱敏值。
- [ ] 明文查看使用独立接口，不和普通查询接口混用。
- [ ] 明文查看接口要求登录、专门权限、租户校验、数据权限校验、操作理由和审计日志。
- [ ] 加密字段查询不使用密文字段 like，需要精确查询时使用标准化 hash 或受控辅助字段。

## 测试
- [ ] Controller 请求层测试覆盖成功路径。
- [ ] Controller 请求层测试覆盖未登录、无权限、跨租户和参数错误。
- [ ] Service 测试覆盖核心业务分支。
- [ ] Mapper 或 SQL 验证覆盖关键查询条件。

## 回归
- [ ] 不影响同模块已有接口。
- [ ] 不影响登录、权限、租户、数据权限、统一响应和异常处理。
- [ ] README、AI 规范、接口文档和权限资源已同步。
```

## 五、多智能体流程

- 主智能体负责最终需求边界、模块归属、实现方案、代码合并、测试结论、提交说明和发布风险判断。
- 子智能体默认只做需求分析、项目学习、问题排查、代码 Review、测试回归和日志分析。
- 子智能体不得自行提交、推送、切分支、修改未分配文件、写死测试租户、写死测试用户、写入私有 IP、写入 token、写入 apiKey 或写入本机绝对路径。
- 多个 worker 只有在写入边界清楚时才能并行，例如一个只改后端接口，一个只改前端页面，一个只补测试。
- 如果多个 worker 可能修改同一个核心 Service、Mapper、SQL 脚本、权限配置、公共工具、认证、租户、数据权限或统一异常，必须由主智能体串行处理。

## 六、子智能体输出要求

- 结论：说明模块归属、接口链路、配置状态或异常原因。
- 证据：提供类名、方法名、SQL、表名、配置文件、日志关键字或行号。
- 风险：说明是否影响权限、租户隔离、数据权限、敏感字段、公共工具、启动配置、调度任务或部署脚本。
- 建议：只提出建议，不替主智能体做最终决策。
